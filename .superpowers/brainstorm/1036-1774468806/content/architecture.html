System Architecture

How the pieces connect — does this match your mental model?

🛵
Keeta Platform
Orders + Auth
🌐
ngrok tunnel
Exposes localhost to Keeta
↓ HTTP POST (webhook)
⚙️
Express Server (Node.js)
POST /webhook — receive orders
GET /events — SSE stream
POST /orders/:id/confirm
POST /orders/:id/ready
POST /orders/:id/cancel
Also calls back to Keeta API to confirm/reject actions
↕ SSE (push) + REST (actions)
🖥️
PDV Browser Interface
Plain HTML + CSS + Vanilla JS
Listens to SSE, shows order cards, sends actions
Flow summary: Keeta sends a new order → ngrok forwards it to Express → Express stores it in memory and pushes it via SSE to the browser → Operator clicks "Confirm" → Express calls Keeta API to confirm the order.
