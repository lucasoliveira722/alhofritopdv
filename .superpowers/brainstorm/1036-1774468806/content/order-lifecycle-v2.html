Cancellation — two sources
🖥️ Operator cancels from PDV
1. Operator clicks "Cancel" on a PLACED or CONFIRMED order
2. PDV sends POST /orders/:id/cancel to Express
3. Express calls Keeta requestCancellation
4. Keeta responds: approved or rejected
5. Order status updated in DB + SSE pushed to browser
🛵 Keeta cancels the order
1. Keeta sends a cancellation event via webhook
2. Express receives it at POST /webhook
3. Express updates order status to CANCELLED in DB
4. SSE pushes update to browser immediately
5. PDV shows the order card as cancelled (no action needed)
↩ Refund Request (Keeta-initiated)
Keeta may send a refund request webhook after delivery. Operator can Accept or Reject from the PDV. Both paths call Keeta API and update the DB.
All state transitions are persisted to PostgreSQL — order status, timestamps for each transition, and the full Keeta event payload are stored for audit and history.
